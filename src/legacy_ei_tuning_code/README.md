# Legacy code — as used for the paper's final subject fittings

This folder is a **frozen snapshot** of the code that produced the final,
complete E/I-tuning fits reported in the paper. It is included **for
reference and provenance only** — to document exactly what was computed,
not as a usable/runnable pipeline.

## Why this exists

At the time of these experiments (September 2024), the fitting pipeline was
built on an experimental fork of [TVB](https://www.thevirtualbrain.org/)
with two custom, unpublished-at-the-time extensions:

- [`tvb-autodiff`](https://github.com/virtual-twin/tvb-autodiff) — JAX
  autodiff backend for TVB
- [`tvb-fit`](https://github.com/virtual-twin/tvb-fit) — generative
  model / fitting utilities

These were moving targets under active development (both pinned to their
`main` branch, not a fixed commit, in the environment definition used at
the time), and `tvb-autodiff`/`tvb-fit` later evolved into
[`tvboptim`](https://pypi.org/project/tvboptim/) (see
[Current alternative](#current-alternative) below), which superseded them.
To make sure the actual dependency code behind the published numbers is
preserved, best-effort source snapshots of both are vendored in
`vendor/` (see [Vendored dependencies](#vendored-dependencies)).

**Do not expect this code to run.** No attempt has been made to update it
for current dependency versions or APIs.

## Provenance

All files in this folder (except this README and the
`generated_run_20240917_130608/` subfolder) are copied verbatim from
commit `50d7ee1` ("Serious fitting run", 2024-09-17) of the original
(private) development repository — the exact commit that generated the
complete 49/49-subject fitting run used for the paper.

That run is reproduced here for reference in
`generated_run_20240917_130608/`: `subjects.txt` and `submit_job.sh` are
the actual artifacts generated by `build_experiment.py` and submitted to
the SLURM cluster (`sbatch submit_job.sh`) for that run. The full raw
fit outputs (`*.pkl` per subject) are not included here to keep this
folder lightweight.

### Pipeline / call graph

1. `build_experiment.py` — generates a timestamped experiment folder,
   picks a global SC scaling factor (99th percentile across subjects),
   writes `subjects.txt`, and generates a SLURM array-job script
   (`submit_job.sh`) that runs `process_subject.py` once per subject.
2. `process_subject.py` — for a single subject: loads SC/FC via
   `utils.load_data_cluster`, builds a TVB `Simulator` using the custom
   `EIB_ReducedWongWangExcInh` model / `EIB_Linear` coupling (defined in
   `EI_RWW_Model.py`), and runs the E/I tuning protocol via `run_EI`
   (defined in `EI_tuning.py`), saving results as a pickle.
3. `EI_tuning.py` — the core E/I-balance tuning loop (FIC calibration +
   iterative FC-based tuning protocol), built on `tvb-autodiff`/`tvb-fit`.
4. `EI_RWW_Model.py` — custom Reduced Wong-Wang excitatory/inhibitory
   balance model and linear coupling class used by the simulator.
5. `utils.py` — data loading (SC/FC) and result pickling helpers.
6. `summarize_fit.py` — post-hoc summary of a batch of per-subject `.pkl`
   fit results (final RMSE / FC-correlation, NaN counts).

### Dependencies

`environment.yaml` is **not** the exact environment used for this run —
no environment file was captured at commit `50d7ee1` itself. It is the
closest available snapshot: the conda environment definition committed
the very next day (2024-09-18) as part of a Snakemake refactor of this
same pipeline, before any dependency changes. It's included as the best
available documentation of what packages/versions this code needed
(notably `tvb-library`, `tvb-framework`, and the `tvb-autodiff`/`tvb-fit`
git forks pinned to their `main` branches at the time).

The SLURM script also references a `EI_TVBC` conda environment and a
cluster-specific working directory
(`/data/cephfs-1/home/users/pillem_c/work/EI_TVBC/ei_tuning_tvbc`) that
are specific to the compute cluster used at the time.

### Vendored dependencies

`vendor/` contains full source snapshots of the two custom TVB extensions
this pipeline depended on (both authored by the same author as this
repository, GPLv3-licensed):

- `vendor/tvb-autodiff@fced0e3` — JAX autodiff backend for TVB
- `vendor/tvb-fit@1a7347d` — generative model / fitting utilities
  (`GenerativeModel`, `GenerativeModelBuilder`, parameters, observation
  models used by `EI_tuning.py`)

**Caveat on the commit choice:** `environment.yaml` pinned both packages
to their `main` branch rather than a fixed commit, and no lockfile/`pip
freeze` from the actual run environment survives. The commits above are
the last commit on `main` at or before the fitting run's timestamp
(2024-09-17 13:05:42) — the best available proxy for what was installed,
not a confirmed exact match. Built docs and notebooks were dropped from
the snapshots to keep this lightweight; `src/` and `tests/` are complete.

## Current alternative

The project has since moved away from TVB + `tvb-autodiff`/`tvb-fit`
entirely. `tvb-autodiff` and `tvb-fit` were themselves later merged into
and superseded by [`tvboptim`](https://pypi.org/project/tvboptim/)
(on PyPI), which is the actively maintained replacement. See the worked
E/I-tuning example in the tvboptim docs:
https://virtual-twin.github.io/tvboptim/workflows/EI_Tuning.html

That current implementation is what should be used for new work; this
folder is historical reference only.
